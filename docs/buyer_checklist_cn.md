# 买家核验清单

卖家运行验机工具后，你会收到一个 ZIP 文件。以下是完整的核验指南，重点以 RTX 4090 为例。

---

## 解压后文件说明

| 文件 | 看什么 |
|------|--------|
| `system_info.txt` | 显卡型号、VRAM、驱动版本、主板、内存 |
| `summary.txt` | 快速摘要 |
| `test_summary.txt` | 各测试通过/跳过/失败状态 |
| `dxdiag_output.txt` | DirectX 诊断，含详细显卡信息 |
| `furmark_log.txt` | FurMark 烤机结果（是否完整跑完） |
| `occt_log.txt` | OCCT 显存测试记录 |
| `hwinfo_sensors.csv` | 传感器时序数据（用 Excel 打开分析温度曲线） |
| `screenshot_*.png` | 测试前中后截图 |

---

## 核验步骤

### 1. 确认显卡型号

打开 `system_info.txt`，找到 `[显卡 GPU]` 部分：

```
名称:       NVIDIA GeForce RTX 4090
显存 (WMI): 24576 MB (24.0 GB)
驱动版本:   31.0.15.6079
PNP设备ID:  PCI\VEN_10DE&DEV_2684...
```

**核验要点：**
- 名称是否与卖家描述一致（RTX 4090 vs RTX 4090D vs RTX 4080 等）
- VRAM 应为 24576 MB（24 GB）
- `VEN_10DE` = NVIDIA（正确），`DEV_2684` = AD102 核心（RTX 4090）
- 驱动版本格式是否正常（过旧驱动版本需注意）

**常见作弊手法：**
- RTX 4090D 改显示名称为 RTX 4090（PNP ID 中 DEV 编号不同）
- GPU-Z 截图比 `system_info.txt` 更难伪造，要求查看 GPU-Z 运行截图

---

### 2. 确认 VRAM 大小（重要）

`system_info.txt` 中 WMI 读取的 VRAM 可能不准（Windows API 限制），建议要求卖家也提供 GPU-Z 截图。

RTX 4090 正确 VRAM：**24 GB GDDR6X**

**红旗：**
- VRAM 显示 16 GB、20 GB 等异常值
- GPU-Z 中 "Memory Size" 与 WMI 数据不一致

---

### 3. FurMark 稳定性

打开 `furmark_log.txt`，检查：

```
测试结果: 通过 (全程稳定运行)
计划时长: 300 秒
实际时长: 301 秒
退出代码: 0
```

**正常：** 测试结果为"通过"，实际时长接近计划时长（300秒），退出代码为 0

**异常：**
- 测试结果为"异常 (提前退出)"
- 实际时长远小于计划时长（卡死或崩溃）
- 退出代码非 0（应用崩溃）

---

### 4. 温度分析（HWiNFO CSV）

用 Excel 或 WPS 打开 `hwinfo_sensors.csv`，查找以下列：
- `GPU Temperature`（GPU 核心温度）
- `GPU Hot Spot Temperature`（结温 / Hot Spot）
- `GPU Fan Speed`（风扇转速）
- `GPU Power`（功耗）

#### RTX 4090 正常范围参考

| 指标 | 正常范围 | 注意 | 异常 |
|------|---------|------|------|
| GPU 核心温度（烤机中） | 70-85°C | 85-95°C | 95°C+ |
| 结温（Hot Spot，烤机中） | 80-100°C | 100-108°C | 108°C+ |
| 风扇转速（烤机中） | 1500-3000 RPM | 3500+ RPM（可能散热问题） | 0 RPM（风扇不转） |
| 功耗（烤机满载） | 300-480W（视功耗墙设置） | <200W（可能功耗限制） | 波动剧烈 |
| 显存温度（GDDR6X） | 80-100°C | 100-108°C | 108°C+ |

**注意：** RTX 4090 GDDR6X 显存体质较热，结温接近 100°C 属正常现象。

---

### 5. OCCT 显存测试

打开 `occt_log.txt` 和对应截图：

**正常：**
- 测试完成，截图中显示 **0 errors**
- 测试时长接近计划时长（600秒 = 10分钟）

**异常：**
- Errors 数量 > 0（显存存在故障颗粒）
- 测试中途崩溃或提前退出

**红旗：** OCCT 报告 1 个以上 error 是严重问题，建议谨慎购买或大幅压价。

---

### 6. 截图核验

检查 `screenshot_*.png` 文件：

1. **before_tests**：测试前桌面状态，确认是正常 Windows 桌面
2. **furmark_前中后**：FurMark 运行时的截图，应能看到甜甜圈渲染画面
3. **occt_start/occt_end**：OCCT 运行界面
4. **occt_result**：OCCT 结束界面，重点看 error 数量

---

## 红旗汇总

以下任意一条出现，建议重新协商价格或拒绝交易：

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| GPU 名称与描述不符 | 严重 | 可能是低端卡改名 |
| VRAM 不是 24 GB | 严重 | 非正品 4090 或显存损坏 |
| FurMark 提前退出 | 严重 | 显卡存在稳定性问题 |
| OCCT 出现 Errors | 严重 | 显存颗粒故障 |
| GPU 温度持续 95°C+ | 中等 | 散热系统可能需要维护 |
| 风扇转速为 0（烤机中） | 严重 | 风扇故障或被禁用 |
| 截图时间和文件时间不匹配 | 可疑 | 可能造假，要求现场视频验机 |
| 功耗远低于正常值 | 中等 | 可能有功耗限制导致性能下降 |
| 驱动版本异常旧 | 低 | 要求更新驱动后重新测试 |

---

## 防伪建议

验机工具可以提高门槛，但无法完全防止造假。建议：

1. **视频验机**：要求卖家开视频，全程观看操作，确认截图是实时生成
2. **现场验机**：条件允许时去卖家处当面验机
3. **查序列号**：GPU-Z 里的序列号（如果显示）可以通过 NVIDIA 官网核实
4. **GPU-Z 验证**：GPU-Z 有"Look Up GPU"功能，可以在线核实参数
5. **交叉验证**：`dxdiag_output.txt` 里的信息应与 `system_info.txt` 一致
6. **保留记录**：妥善保存验机 ZIP，作为日后维权的凭证

## 关于 RTX 4090 vs RTX 4090D 区分

RTX 4090D 是专为中国大陆市场发行的出口管制版本（削减 CUDA 核心）：

| 特征 | RTX 4090 | RTX 4090D |
|------|----------|-----------|
| CUDA 核心 | 16384 | 14592 |
| GPU 核心 | AD102-300 | AD102-301 |
| PNP DEV ID | 2684 | 2685 |
| 性能差异 | 基准 | 约低 11% |
| 功耗 | 450W TDP | 425W TDP |

在 `system_info.txt` 的 `PNP设备ID` 中，`DEV_2684` = 4090，`DEV_2685` = 4090D。如果卖家将 4090D 标价为 4090，这是价格欺诈。
